Record Property

Record Property ist ein CSS-Plugin, welches einen Recordnamen aus einem Textfeld oder Kontextmenu eines anderen CSS-Plugins liest und sämtliche für diesen Record zur Verfügung stehende Informationen darstellt.

Basic logic

  • Verbinden mit RDB (Oracle)
    • wenn es geht
      • hole Record-Typ von DAL (oder von Recordname, wenn DAL schiefgeht)
      • hole Daten von RDB, fülle erste und zweite Spalte
      • hole Daten von DAL, fülle dritte Spalte
      • hole Daten von RMI (IOC), fülle vierte Spalte (Achtung: normalerweise leer), fülle vierte Spalte
    • wenn es nicht geht
      • hole Record-Typ von DAL (oder von Recordname, wenn DAL schiefgeht)
      • hole Feldnamen für diesen Typ von RDB , fülle erste Spalte, zweite Spalte bleibt leer
      • hole Daten von DAL, fülle dritte Spalte
      • hole Daten von RMI (IOC), fülle vierte Spalte (Achtung: normalerweise leer), fülle vierte Spalte
auf RMI(IOC) wird über das SaveButton-Plugin zugegriffen.

Ist der Recordname nicht gültig?

Das Plugin löscht den letzten Teil des Namens, wenn darin "." vorkommt und der Name kürzer als 6 Zeichen ist. (Beispiel:".abcd")

Wenn der Recordname überhaupt nicht gefunden wird, zeigt das Plugin nur eine leere Tabelle.

Keine Daten da?

Wenn die Daten nicht verfügbar oder für eine einzelne Zelle nicht zugreifbar sind, wird die Meldung "N/A" auf einem grau-gelben Hintergrund ausgegeben.

Wenn die Daten für die Zelle, die geholt wurde(n), leer sind, wird nicht die "N/A"-Meldung ausgegeben, sondern ein leere Zeichenkette.

Tips und Tricks

Um einen Feldnamen schnell zu finden, muß man ihn nur tippen, dann wird automatisch auf die Zeile mit diesem Feldnamen gesprungen.

Spaltennamen können in der Datei "messages.properties" leicht geändert werden.
