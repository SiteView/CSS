Alarm Tree User's Guide

Dies ist der User-Guide für Alarm-Tree.

Preferences

Die Preference-Page für den Alarm-Tree ist CSS Applications > Alarm > Alarm Tree.

Zuerst sicherstellen, dass der LDAP-Directory-Server und die JMS-Server richtig aufgesetzt sind. Aus Redundanzgründen lauscht der Alarm-Tree auf zwei JMS-Server. Wenn man die JMS Settings ändert, nachdem der Tree schon läuft, ist ein Restart des Trees erforderlich.

Nach Aufsetzten der Server kann man die Facility names list benutzen, um die Facilities hinzuzufügen, die man im Tree angezeigt haben möchte. Man kann eine beliebige Anzahl von Facilities auswählen. Man kann die LDAP Settings und die Facility-Auswahl sogar ändern, wenn der Tree schon läuftt. Einfach auf den "Reload"-Button in der Tree-Toolbar klicken, nachdem die neuen Preferences gesichert sind. Der Alarm-Tree benutzt dann ab dem Reload die neuen Settings.

Using the view

Icons

Die View zeigt jeden Knoten mit farbigen Kreisen an, welche den aktuellen Alarmzustand abbilden, und, wenn zutreffend, den schwerwiegensten Alarm, der noch nicht quittiert ist. Die Farben haben die folgende Bedeutungen:

  • Blau: invalid
  • Rot: major
  • Gelb: minor
  • Grün: alarm

Das Icon rechts im Vordergrund zeigt immer den aktuellen Alarmzustand an. Wenn der aktuelle Alarm quittiert wurde, wird oben im Alarm-Icon ein Häkchen eingeblendet. Wenn es für den Knoten alte Alarme gibt, die höheres Gewicht als der aktuelle Alarm haben und die noch nicht quittiert sind, wird ein zweites Icon links hinter dem primären Icon ausgegeben.

Für die Blatt-Knoten, die individuelle Records (Prozess-variable) darstellen, zeigt das Icon den Alarmzustand dieses Records an. Für Knoten, die Child-Knoten haben, zeigt das Icon den höchsten Alarmzustand seiner Kinder.

Acknowledging alarms

Wenn ein Knoten unquittierte Alarmzustände enthält, kann man das Kontextmenu des Knotens benutzen, um Quittungsmeldungen für diesen Knoten zu senden. Man beachte, dass der Knoten nicht sofort als quittiert dargestellt wird. Der Tree aktualisiert den Zustand des Knotens, sobald er die Quittungsmeldung empfangen hat.

Eine Quittierung quittiert immer sowohl den aktuellen Alarm, als auch alle vorherigen unquittierten Alarme. Wenn der ausgewählte Knoten ein Parent-Knoten ist, wird für jedes Child, welches unquittierte Alarme hat, eine Quittungsmeldung gesendet.

Opening the alarm display

Falls ein Alarm-Display für einen Record im LDAP-Directory konfiguriert ist, kann man dieses Alarm-Display aus dem Kontextmenu mit Hilfe der "Run alarm display"-Action öffnen. Man beachte, dass dieses Menu-Item nur erscheint, wenn ein einzelner Blatt-Knoten ausgewählt ist. Es ist nicht verfügbar für Parent-Knoten oder Vielfach-Auswahlen. Wenn der Menu-Item ausgegraut erscheint, ist für diese Prozessvariable kein Alarm-Display konfiguriert, oder das Attribut 'value' sieht nicht wie ein gültiger CSS-Display-Dateiname aus (Dateiname muss mit .css-sds enden).

Editing Support

Die View enthält Unterstützung zum Editieren des LDAP-Directories, welches die Struktur des Knotens enthält, der in der View dargestellt wird.

Using the context menu

Das Kontextmenu für einen Knoten enthält Menu-Items zum Löschen des Knotens. Für Knoten, die keine Channels sind, enthält es auch Menu-Items zum Erzeugen einer neuen Komponente oder Subkomponente unterhalb des ausgewählten Knotens und zum Erzeugen eines neuen Channels innerhalb des ausgewählten Knotens. Ob eine Komponente oder Subkomponente erzeugt wird, wird automatisch bestimmt: Wenn der ausgewählte Knoten ein Facility ist, kann man Kompnenten-Knoten erzeugen; wenn er eine Komponente oder Subkomponente ist, kann man Subkomponenten erzeugen.

Man beachte, dass das Löschen nicht rekursiv arbeitet, d.h. man kann nur Knoten löschen, die keine Subknoten enthalten. Das liegt einerseits an den Einschränkungen des LDAP-Servers und schützt andereseits gegen zufälliges Löschen grosser Trees.

Using drag&drop

Zusätzlich zur Benutzung des Kontextmenus kann man auch die drag&drop -Features des Control-System-Studios benutzen, um weitere Channels zum Tree hinzuzufügen. Einfach in irgendeiner View, die drag&drop unterstützt, einen oder mehrere Channels auswählen und in den Alarm-Tree reinziehen. Während man Items über den Tree zieht, ändert sich der Maus-Cursor, um anzuzeigen, wo man die Channels loslassen kann.

Man beachte, dass der Tree z.Zt. nur unterstützt, Items in den Tree zu ziehen. Man kann nicht Channel-Namen oder andere Knoten aus dem Tree heraus in andere Views ziehen. Diese Möglichkeiten werden wohl in einer zukünftigen Version verfügbar sein.

Editing the properties of nodes

Man kann die Eclipse Property View benutzen, um die Eigenschaften der Knoten im Tree zu editieren. Die Eigenschaften werden als Attribute im LDAP-Verzeichnis gespreichert. Die Property View kann durch Anklicken des "Property View"-Buttons in der Toolbar der View geöffnet werden.

Folgende Eigenschaften sind verfügbar

Alarm display
Der Dateiname des Alarm-Displays, das mit dem Knoten verknüpft ist. Wenn diese Eigenschaft den Namen einer CSS-SDS-Display-Datei enthält, kann man das Display aus dem Kontextmenu im Alarm-Tree öffnen.
Display
Der Dateiname des Displays, das mit dem Knoten verknüpft ist. Wenn diese Eigenschaft den Namen einer CSS-SDS-Display-Datei enthält, kann man das Display aus dem Kontextmenu im Alarm-Tree öffnen.
Help guidance
Eine Kurzbeschreibung des Knotens.
Help page
URL einer WEB-Seite, welche weitergehende Information über den Knoten enthält.
Name
Der Name des Knotens. Diese Eigenschaft kann nicht editiert werden.
Object class
Die Objektklasse des Eintrags im LDAP-Verzeichnis. Diese Eigenschaft kann nicht editiert werden.
Strip chart
Der Dateiname einer Strip-Chart-Datei, die mit dem Knoten verknüpft ist. Wenn diese Eigenschaft gesetzt ist, kann die Datei vom Kontextmenue aus geöffnet werden.

Man beachte, dass Dateinamen (Pfade) relativ zum CSS-Workspace gesetzt sein müssen. Besipiel: Der Dateiname "/SDS/displays/example.css-sds" zeigt auf die Datei "example.css-sds" im Unterverzeichnis "displays" welches im "SDS"-Projekt des Workspace liegt.
