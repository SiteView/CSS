Alarm Tree User's Guide

This is the user's guide for the alarm tree.

Preferences

The preference page for the alarm tree is CSS Applications > Alarm > Alarm Tree.

First, make sure that the LDAP directory server and the JMS servers are set up correctly. The alarm tree will listen to two JMS servers for redundancy. Changing the JMS settings after the tree is already running requires a restart.

After the servers are set up, you can use the Facility names list to add the facilities that you would like to display in the tree. You can select any number of facilities. You can change the LDAP settings and facility selection even when the tree is already running. Simply click the "Reload" button in the tree's toolbar after saving the new preferences. The alarm tree will use the new setting when reloading the tree.

Using the view

Icons

The view displays each node with colored circles which indicate the current alarm state and, if applicable, the most severe past alarm which has not been acknowledged yet. The colors have the following meaning:

  • Blue: invalid
  • Red: major
  • Yellow: minor
  • Green: no alarm

The right/foreground icon always indicates the current alarm state. If the current alarm is acknowledged, a checkmark is displayed on top of the alarm icon. If there are past alarms for a node which have a higher severity than the current alarm and which have not been acknowledged yet, a second icon is displayed to the left of and behind the primary icon.

For the leaf nodes which represent individual records (process variables), the icon indicates the alarm state of that record. For nodes which have child nodes, the icon indicates the highest severity of its children.

Acknowledging alarms

When a node has any unacknowledged alarms, you can use the context menu of then ode to send an acknowledgement message for that node. Note that the node will not be displayed as acknowledged immediately. The tree will update the node's state when it receives the acknowledgement message.

An acknowledgement always acknowledges both the current alarm, and any past unacknowledged alarms. When the selected node is a parent node, one acknowledgement message will be sent for each child that has an unacknowledged alarm.

Opening the alarm display

If an alarm display is configured for a record in the LDAP directory, you can open that alarm display from the context menu using the "Run alarm display" action. Note that this menu item appears only if a single leaf node (process variable node) is selected. It is not available for parent nodes or multi selections. If the menu item appears grayed out, no alarm display is configured for that process variable, or the attribute value does not look like a valid CSS display file name (the value must end with .css-sds).

Editing Support

The view contains support for editing the LDAP directory which defines the structure of the node displayed in the view.

Using the context menu

The context menu for a node contains menu items for deleting the node. For nodes which are not channels, it also contains menu items for creating a new component or subcomponent below the selected node, and for creating a new channel within the selected node. Whether a component or a subcomponent is created is determined automatically: if the selected node is a facility, you can create component nodes, if it is a component or a subcomponent, you can create subcomponents.

Note that deletion does not work recursively, i.e., you can only delete nodes that do not have any subnodes. This is because of restrictions of the underlying LDAP server and also to prevent accidental deletion of large trees.

Using drag&drop

In addition to using the context menu, you can also use the drag&drop features of the Control System Studio to add additional channels to the tree. Simply select one or more channels in any view that supports drag&drop and drag them into the alarm tree. While you are dragging items over the tree, the mouse cursor will change to indicate where you can drop those items.

Note that the tree currently only supports dragging items into the tree. You cannot drag channel names or other nodes out of the tree into other views. Support for dragging will probably be added in a later version.

Editing the properties of nodes

You can use the Eclipse Property View to edit the properties of the nodes in the tree. The properties are stored as attributes in the LDAP directory. The property view can be opened by clicking the "Property View" button in the view's toolbar.

The available properties are:

Alarm display
The filename of the alarm display associated with the node. If this property contains the name of a CSS-SDS display file, you can open the display from the context menu in the Alarm Tree.
Display
The filename of the display associated with the node. If this property contains the name of a CSS-SDS display file, you can open the display from the context menu in the Alarm Tree.
Help guidance
A short description of the node.
Help page
URL of a web page which provides further information for the node.
Name
The name of the node. This property cannot be edited.
Object class
The object class of the entry in the LDAP directory. This property cannot be edited.
Strip chart
The filename of a strip chart file associated with the node. If this property is set, the file can be opened from the context menu.

Note that filenames must be relative to your CSS workspace. For example, the filename "/SDS/displays/example.css-sds" points to the file "example.css-sds" in the "displays" folder contained in the "SDS" project in your workspace.
