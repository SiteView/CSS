Overview

Diles stands for Digital Logic Editor & Simulator. Diles allows users to create visual logic diagrams using different logic operators such as AND, OR, XOR.

Diles is based on Eclipse Graphical Editing Framework (GEF).

Usage

Diles allows user to create logic diagrams using digital widgets. Those widgets can be connected. Widgets with inputs will read output of upper widget. If upper widget is after the lower widget in the time line, the default value (false) will be taken as input. In the next cycle, after upper widget has been executed, the value from a cycle before will be taken as input.

TDE & TDD Timers

TDE Timer waits for a certain amount of seconds if the input is 0. It always outputs 1.

TDD Timer waits for a certain amount of seconds if the input is 1. It always outputs 0.

Inputs and Outputs

There are two kinds of inputs (Hardware Input and Command) and two kinds of outputs (Hardware Output and Status).

Both inputs act the same way in Diles. Same goes for outputs. The reason there are two implemented is for the user to be able to know which input (or output) is which.

Analog Input and Comparator

Analog Input has a value of some Real number. Comparator takes two Analog Input widgets for input. If + input is larger or same than -, Comparator will return true. If + input is smaller than -, Comparator will return false.

Connection

Connection connects widgets together. It can start only in output and end in input. Only one input connection is possible per one widget input terminal. More output connections are possible per widget output terminal.

Connections can be bent. This is a visual feature of connections.

Possible integration with IOC

As far as I know, IOC can take some code as parameter. To be possible to develop this application further so it would be completely integrated with IOC, a program has to be written that would transform XML code (which is located in .diles) into IOC code. The program should use some external XML helper (as JDOM) to get the right data.

The relevant data for IOC would be:

-- Activity type, unique_id, column, name, status -- (last two are only relevant for inputs and outputs)

-- connection source_id, source_terminal, target_id, target_terminal -- (the last one could be ignored, as there is only one output terminal per widget possible

The program's job would also be to create temporary variables, as they are not used in Diles.
